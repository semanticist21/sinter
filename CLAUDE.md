## Project Overview

**Sinter** is a high-performance image compression library combining Rust WASM with TypeScript. It reduces image file sizes while maintaining quality and optionally preserving EXIF metadata. Supports JPEG, PNG, WebP, AVIF formats.

**Core API**: `src/index.ts` exports `CompressImage(file, options)` → `Promise<File>`
**Language**: Hybrid Rust (WASM) + TypeScript

## Architecture

### Structure

```
sinter/
├── wasm/                       # Rust WASM implementation
│   ├── Cargo.toml
│   ├── src/
│   │   ├── lib.rs                  # compress_image() entry point; returns CompressResult
│   │   ├── format.rs               # ImageFormat enum: MIME detection + magic bytes + metadata
│   │   ├── formats/                # Format-specific encode/decode
│   │   │   ├── mod.rs              # Format dispatcher (decode/encode APIs)
│   │   │   ├── jpeg.rs             # JPEG decode + encode + quality normalization
│   │   │   ├── png.rs              # PNG decode + encode + quality normalization
│   │   │   ├── webp.rs             # WebP decode + encode → PNG fallback
│   │   │   └── avif.rs             # AVIF decode + encode + quality normalization
│   │   ├── resize.rs               # Aspect-ratio-preserving Lanczos3 resize
│   │   ├── exif.rs                 # EXIF extraction/insertion
│   │   ├── error.rs                # Error types
│   │   └── constants.rs            # MAX_DIMENSION, SUPPORTED_FORMATS
│   └── pkg/                        # Generated by wasm-pack (auto-linked)
│
├── src/                        # TypeScript wrapper + public API
│   ├── index.ts                        # Public CompressImage() API + Worker management
│   ├── compress.worker.ts              # Worker script (WASM execution)
│   ├── utils.ts                        # convertToBytes() only (byte unit conversion)
│   ├── __tests__/                      # Integration tests for public API
│   └── (tsconfig.json, biome.json included in project root)
│
├── rslib.config.ts             # Library build config (ESM + CJS)
├── package.json                # Dependencies + build scripts
├── tsconfig.json               # Strict TypeScript + declaration-only emit
└── biome.json                  # Formatting: 2-space, 100-char line width
```

### Design Principles

1. **Single Responsibility**: Rust handles all image ops (decode→resize→encode→EXIF), TypeScript wraps File I/O
2. **Aspect Ratio**: Always preserved; both maxWidth & maxHeight applied independently
3. **Quality Normalization**: 0-100 scale internally mapped per format
4. **Dual Distribution**: Rslib generates ESM (browser) + CommonJS (Node.js)
5. **Linked WASM**: Auto-detected during `bun run dev`
6. **Non-Blocking UI**: WASM compression runs in Worker thread, keeps main thread responsive

   → Builds TypeScript + imports linked WASM
   → Outputs to dist/:
     - index.js (ESM, minified)
     - index.cjs (CommonJS, minified)
     - index.d.ts (TypeScript declarations)

**Distribution artifact (dist/):**
- Publishable to npm (files: ["dist"] in package.json)
- Includes WASM binary inlined in JS bundles (via wasm-pack)
- Tree-shakeable ESM exports
- Must be committed to git (required for npm publish)

- Format-specific quality ranges:
  - JPEG: 0-100 (direct pass-through)
  - PNG: 0-9 (mapped from 0-100)
  - AVIF: 0-100 (ravif encoder)

| Feature | Behavior |

| **EXIF** | Preserved by default unless `preserveExif: false` |
| **Format** | Uses input MIME type if unspecified; defaults to JPEG |
| **Output Name** | Original name + "-compressed" + format extension |
| **WebP** | Falls back to PNG (libwebp-sys incompatibility) |

## Execution Flow

When `CompressImage(file, options)` is called:

```
1. TypeScript Main Thread (src/index.ts)
   - Read file → ArrayBuffer
   - postMessage to Worker (non-blocking)
   - Return Promise

2. Worker Thread (src/compress.worker.ts) [async]
   - Receive message from main thread
   - Call Rust WASM (blocking on worker, not main)

3. Rust WASM (wasm/src/) [worker thread]
   - extract EXIF (if preserveExif=true)
   - decode image
   - resize (aspect ratio preserved)
   - encode (quality=85)
   - quality loop (reduce by 5 if maxSize violated)
   - insert EXIF back
   - return compressed data

4. Worker Thread
   - postMessage result to main thread

5. TypeScript Main Thread Result
   - Create File object with correct MIME type
   - Resolve Promise<File>
```

**Key**: WASM processing happens on worker thread, main thread stays responsive for UI.

### Implementation Notes

- **Worker Thread Architecture**:
  - Compression runs on dedicated worker thread (`compress.worker.ts`), main thread stays responsive
  - Worker is singleton instance (`getCompressWorker()` reuses same instance across calls)
  - Message-based communication with unique IDs for tracking concurrent requests
  - Used to prevent blocking main thread UI during WASM execution

- **Data Flow Separation**:
  - `src/index.ts` (CompressImage) handles: File reading, Worker lifecycle, File object creation from Rust metadata
  - `src/compress.worker.ts` handles: Calling Rust compress_image(), passing CompressResult back
  - TypeScript receives MIME type and extension from Rust (Single Source of Truth)

- **Format Detection** (in `wasm/src/format.rs`):
  - MIME type detection via `ImageFormat::from_mime_type()` handles `image/jpeg`, `image/jpg`, `image/png`, etc.
  - Magic bytes fallback via `ImageFormat::detect_from_bytes()` for JPEG (FF D8 FF), PNG (89 50 4E 47), WebP (RIFF...WEBP), AVIF (ftyp)
  - `ImageFormat::detect()` tries MIME first, then magic bytes if MIME fails
  - Rust returns both format enum and metadata (MIME type, extension) to TypeScript

- **Quality Normalization**: Happens in each format module's encode function:
  - JPEG: 0-100 (direct pass-through)
  - PNG: Maps 0-100 → 0-9 compression level
  - AVIF: 0-100 (ravif encoder specific)
  - WebP: Falls back to PNG due to libwebp-sys native compilation limitation in WASM

- **Quality Fallback Loop**: If output exceeds maxSize, quality reduced automatically:
  - Start: quality=85
  - Loop: quality -= 5 until file ≤ maxSize or quality=20
  - Controlled in WASM `lib.rs` compress_image_internal loop

- **EXIF Handling**:
  - Extracted before processing (if preserveExif=true)
  - Inserted back after encode without re-encoding
  - Uses img-parts library to avoid re-compressing image data

- **Build Optimizations**:
  - Cargo release builds: aggressive opts (opt-level="z", LTO)
  - wasm-pack: web target with minification
  - rslib: ESM + CJS bundles, tree-shakeable exports

- **Memory Limits**:
  - MAX_DIMENSION = 4096px (browser memory constraint)
  - ~300MB memory for 16MP image at 4096px
  - Larger dimensions risk browser OOM; increase cautiously

## Development Patterns

### Adding a New Image Format

1. Create `wasm/src/formats/[format].rs` with:
   - `pub fn decode(data: &[u8]) -> SinterResult<Image>` - decodes bytes → Image struct
   - `pub fn encode(image: &Image, quality: u8) -> SinterResult<Vec<u8>>` - encodes Image → bytes with quality normalization

2. Update `wasm/src/formats/mod.rs`:
   - Import new format module
   - Add dispatch logic in `decode()` and `encode()` functions

3. Update `wasm/src/constants.rs`:
   - Add format to SUPPORTED_FORMATS constant

4. Test with: `bun run check` (catches Rust errors early)

5. Format-specific quality mapping (in encode function):
   - Define how 0-100 quality scale maps to your encoder's range
   - Example (PNG): `compression_level = ((100 - quality) / 10) as u8` maps to 0-9 PNG compression

### Modifying Compression Quality Behavior

Core quality logic is in `wasm/src/lib.rs` (`compress_image_internal` function):
- Default quality: 85
- Fallback loop: quality -= 5 each iteration until file ≤ maxSize
- Minimum quality: 20

To adjust:
1. Change initial quality: Update `quality = 85` default in lib.rs
2. Change fallback step: Update `quality -= 5` in loop
3. Change minimum: Update loop condition `quality >= 20`
4. Change format-specific: Edit quality normalization in `wasm/src/formats/[format].rs`

### Testing Compression Logic

Use `src/__tests__/` for integration testing:
- Tests call public `CompressImage()` API
- Validates full workflow including File I/O, Worker communication, and metadata handling
- Verifies MIME type and extension from Rust are correct
